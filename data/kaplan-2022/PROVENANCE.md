## What's here

`omino/{n:02}omino_{Hmin}up.txt` for n âˆˆ {7, â€¦, 19}. The `Hmin` suffix is the
lowest Heesch number included in that file (Kaplan's naming convention):

- `0up` for n â‰¤ 10 â€” every non-tiling polyomino, Hc â‰¥ 0.
- `2up` for n â‰¥ 11 â€” only the rare Hc â‰¥ 2 shapes; Hc âˆˆ {0, 1} are too numerous
  to enumerate exhaustively at those sizes.

File format: pairs of lines per shape.

```
1 0 1 1 0 2 1 2 1 3 2 3 3 3
Hc = 1 Hh = 1
```

Line 1 is the cell coordinates (one (x, y) per pair, integer). Line 2 records
the published `Hc` (hole-free Heesch number) and `Hh` (Heesch number with
holes permitted in the outer corona).

This format is a human-readable annotation **distinct from `sat`'s own
input/output format**; the regression harness in `benchmarks/kaplan/`
converts between them.
